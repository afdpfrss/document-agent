## 1. 適用範囲と定義
<!-- section_id: sec_1 -->

### 第1条(目的)
本規程は、株式会社サンプル(以下「当社」という。)が事業活動において取り扱う個人情報について、個人情報の保護に関する法律(以下「個人情報保護法」という。)、行政手続における特定の個人を識別するための番号の利用等に関する法律(以下「マイナンバー法」という。)、EU一般データ保護規則(以下「GDPR」という。)、その他関連法令、ガイドライン(個人情報保護委員会「個人情報の保護に関する法律についてのガイドライン」、JIS Q 15001:2023、ISO/IEC 27701:2019)を遵守し、個人情報の適正な取扱いと本人の権利利益の保護を図ることを目的とする。

### 第2条(適用範囲)
本規程は、当社の全事業所、全役職員(役員、正社員、契約社員、嘱託、派遣社員、パートタイマー、アルバイト)、並びに業務委託先(再委託先を含む)に適用する。本規程の対象となる個人情報の本人(以下「本人」という。)には、お客様、見込み顧客、お取引先様の役職員、当社の役職員及びその家族、採用応募者、退職者、株主、その他あらゆる個人が含まれる。

### 第3条(用語定義)
本規程における主要な用語は、個人情報保護法の定義に従い、以下のとおりとする。

| 用語 | 定義 |
|------|------|
| 個人情報 | 生存する個人に関する情報であって、当該情報に含まれる氏名、生年月日その他の記述等により特定の個人を識別できるもの(他の情報と容易に照合でき、それにより特定の個人を識別できるものを含む)及び個人識別符号が含まれるもの。 |
| 個人識別符号 | DNA配列、顔認識データ、指紋、声紋等の身体的特徴を変換した符号、又はマイナンバー、旅券番号、運転免許証番号等の公的番号。 |
| 個人データ | 個人情報データベース等を構成する個人情報。 |
| 保有個人データ | 当社が開示、訂正、利用停止等の権限を有する個人データ。 |
| 要配慮個人情報 | 人種、信条、社会的身分、病歴、犯罪歴、犯罪被害事実、その他本人に対する不当な差別、偏見その他の不利益が生じないようにその取扱いに特に配慮を要するものとして政令で定める記述等が含まれる個人情報。 |
| 仮名加工情報 | 他の情報と照合しない限り特定の個人を識別できないように個人情報を加工して得られる個人に関する情報。 |
| 匿名加工情報 | 特定の個人を識別できないように個人情報を加工し、当該個人情報を復元できないようにした個人に関する情報。 |
| 個人関連情報 | 生存する個人に関する情報であって、個人情報、仮名加工情報、匿名加工情報のいずれにも該当しないもの(Cookie ID、IPアドレス、閲覧履歴等)。 |

### 第4条(域外法令の適用)
当社が以下に該当する個人データを取り扱う場合、当該域外法令も適用する。
- EU/EEA域内に所在する個人に対する商品・サービス提供又は行動監視に伴う個人データ:GDPR
- 英国居住者の個人データ:UK GDPR、Data Protection Act 2018
- カリフォルニア州住民の個人情報:CCPA/CPRA
- 中国国内で収集された個人情報:中華人民共和国個人情報保護法(PIPL)

### 第5条(個人情報保護体制)
| 役割 | 任命 | 責任 |
|------|------|------|
| 個人情報保護管理者(CPO) | 法務部長 | 当社の個人情報保護全体統括、個人情報保護委員会との窓口 |
| 個人情報保護事務局 | 法務部コンプライアンス課 | 規程整備、教育、苦情対応、開示請求対応 |
| EU代理人 | 外部法律事務所(別途指定) | GDPR第27条に基づくEU域内代理人 |
| 部門個人情報管理責任者 | 各部長 | 所管部署の個人情報取扱業務の管理 |
| 利用者(全役職員) | ─ | 規程遵守、漏えい時の即時報告 |

## 2. 取得と利用
<!-- section_id: sec_2 -->

### 第6条(取得の原則)
当社は、個人情報を取得するにあたり、以下を遵守する。
1. 適法かつ公正な手段によることとし、偽りその他不正の手段による取得を行わない。
2. 取得に際しては、利用目的を特定し、本人に通知又は公表する。
3. 本人から直接書面又は電磁的方式により取得する場合は、利用目的を明示する。
4. 要配慮個人情報の取得には、原則として本人の事前同意を得る(法令に基づく場合等の例外を除く)。
5. マイナンバーは、法定の利用範囲(社会保障、税、災害対策)に限定して取得する。

### 第7条(利用目的の特定及び公表)
当社は、業務カテゴリごとに利用目的を特定し、プライバシーポリシー(コーポレートサイト掲載)及び各種申込書面において明示する。代表的な利用目的は以下のとおりとする。
- 商品・サービスの提供、契約の履行、アフターサービス
- 代金請求、入金確認、与信判断
- 商品・サービスに関する案内、アンケート、マーケティング分析
- お問い合わせ・苦情への対応
- 採用選考、入退社手続、人事労務管理、給与計算、社会保険手続
- 株主管理、株主総会運営、配当金支払
- 法令上の義務履行、紛争対応

### 第8条(利用目的の変更)
特定した利用目的を超えて個人情報を利用する場合は、原則として本人の事前同意を得る。利用目的の変更は、変更前の利用目的と関連性を有すると合理的に認められる範囲内でなければならない。

### 第9条(本人同意の取得)
本人同意は、本人が利用目的等を理解した上で自らの意思で同意を表明していることが明確であるよう、能動的な行為(チェックボックスへの自発的チェック、署名等)によって取得する。ダークパターン(本人を誤認させるUI)による同意取得を禁止する。

### 第10条(オンライン識別子・Cookie等)
Webサイト、モバイルアプリ等におけるCookie、ローカルストレージ、デバイス識別子、IPアドレス等の取得は、同意管理プラットフォーム(CMP)により本人の選択を取得・記録する。本人は、必須Cookie以外について、いつでも同意を撤回できる。

### 第11条(データ最小化と保管期間)
業務目的の達成に必要な最小限の個人情報のみを取得し、保管する。保管期間は業務カテゴリごとに「個人情報取扱業務一覧表」に定め、当該期間経過後は遅滞なく消去又は匿名化する。

## 3. 第三者提供と委託
<!-- section_id: sec_3 -->

### 第12条(第三者提供の原則禁止)
当社は、個人情報保護法第27条の規定に従い、あらかじめ本人の同意を得ることなく個人データを第三者に提供しない。ただし、以下の場合はこの限りでない。
1. 法令に基づく場合
2. 人の生命、身体又は財産の保護のために必要がある場合で本人の同意を得ることが困難な場合
3. 公衆衛生の向上又は児童の健全育成の推進のために特に必要がある場合で本人の同意を得ることが困難な場合
4. 国の機関等への協力が必要な場合
5. 学術研究機関等への提供(個人情報保護法第27条第1項第5〜7号)

### 第13条(オプトアウト方式による提供)
当社は原則としてオプトアウト方式による第三者提供を行わない。仮に実施する場合は、個人情報保護委員会への事前届出、本人通知又は公表、容易な停止手段の確保を行う。要配慮個人情報、不正取得した個人データ、オプトアウトにより取得した個人データはオプトアウト方式での再提供を禁止する。

### 第14条(委託先の選定及び監督)
個人データの取扱いを委託する場合、以下の手順を遵守する。
1. 選定基準(セキュリティ体制、認証取得状況、過去の事故歴等)に基づき委託先候補を評価する。
2. 個人情報取扱業務に関する委託契約(当社雛形)を締結する。契約には、目的外利用禁止、再委託の事前承認、安全管理措置、漏えい時報告義務、契約終了時の返却・消去、監査受入義務、損害賠償を含める。
3. 委託先に対し、年1回以上のセキュリティチェック(質問票回答、必要に応じ実地監査)を実施する。
4. 再委託は当社の事前書面承認を必要とし、再委託先にも同等の義務を課す。

### 第15条(共同利用)
個人データを特定の者と共同利用する場合、個人情報保護法第27条第5項第3号に基づき、共同利用する旨、項目、範囲、利用目的、管理責任者を本人に通知又は公表する。

### 第16条(外国にある第三者への提供)
外国にある第三者に個人データを提供する場合、原則として本人の同意を取得する。同意取得時には、移転先国名、当該国の個人情報保護制度、移転先における安全管理措置に関する情報を本人に提供する。同意以外の根拠(基準適合体制、十分性認定)による移転は、CPOの承認を要する。EU/EEA域内からの域外移転については、十分性認定、標準契約条項(SCC)、拘束的企業準則(BCR)等GDPR適合根拠を整備する。

### 第17条(個人関連情報の提供)
個人関連情報を第三者に提供し、当該第三者が個人データとして取得することが想定される場合は、当該第三者が本人同意を取得していることを確認しなければならない(個人情報保護法第31条)。

### 第18条(仮名加工情報・匿名加工情報の取扱い)
仮名加工情報・匿名加工情報を作成する場合、個人情報保護委員会規則に従い加工基準を遵守し、識別行為の禁止、安全管理措置、加工方法等情報の安全管理を行う。匿名加工情報の作成・第三者提供時には、項目及び提供方法を公表する。

## 4. 開示等請求対応
<!-- section_id: sec_4 -->

### 第19条(本人からの請求権)
本人(又はその代理人)は、当社に対し、保有個人データに関し以下の請求を行うことができる。
1. 利用目的の通知請求(個人情報保護法第32条)
2. 開示請求(電磁的記録による開示請求を含む。同法第33条)
3. 訂正、追加又は削除請求(同法第34条)
4. 利用停止又は消去請求(同法第35条第1項・第5項)
5. 第三者提供の停止請求(同法第35条第3項・第5項)
6. 第三者提供記録の開示請求(同法第33条第5項)

GDPR・PIPL等が適用される場合は、アクセス権、訂正権、消去権(忘れられる権利)、処理制限権、データポータビリティ権、異議申立権、自動意思決定に関する権利等の請求にも対応する。

### 第20条(請求窓口)
請求は以下の窓口にて受け付ける。
- 専用窓口メールアドレス:privacy@sample.example
- 専用ダイヤル:03-XXXX-XXXX(平日9:00〜17:00)
- 郵送:〒XXX-XXXX 東京都港区●●●● 株式会社サンプル 個人情報保護事務局
- WebフォームURL:https://www.sample.example/privacy/request

### 第21条(請求受付手順)
1. 請求書面(当社所定様式又はこれに準ずる書面)及び本人確認書類(運転免許証、マイナンバーカード(マイナンバー部分は隠す)、健康保険証等のうち1点。代理人請求の場合は委任状及び代理人の本人確認書類)を受領する。
2. 本人確認、請求対象データの特定を行う。
3. 個人情報保護事務局が原則として2週間以内に対応方針を決定し、本人に通知する。
4. 対応完了後、対応結果を書面又は電磁的方式により本人に通知する。
5. 開示請求に対しては、原則として手数料1件1,000円(税別)を徴収する。利用停止請求等は無料とする。

### 第22条(請求拒否事由)
以下のいずれかに該当する場合は、請求の全部又は一部に応じないことがある。その場合、理由を付して本人に通知する。
1. 本人又は第三者の生命、身体、財産その他の権利利益を害するおそれがある場合
2. 当社の業務の適正な実施に著しい支障を及ぼすおそれがある場合
3. 他の法令に違反することとなる場合
4. 本人確認ができない場合

### 第23条(苦情対応)
当社は、個人情報の取扱いに関する苦情を上記窓口にて受け付け、適切かつ迅速に対応する。苦情内容、対応経過、対応結果は記録し3年間保管する。当社にて解決困難な場合、認定個人情報保護団体(該当する場合)又は個人情報保護委員会の相談窓口を案内する。

### 第24条(安全管理措置)
当社は、個人データの漏えい、滅失又は毀損の防止その他の安全管理のため、以下を講じる。
- **組織的安全管理措置**:管理体制構築、個人データ取扱状況の把握、漏えい等対応体制(本規程及びdoc_031参照)、点検
- **人的安全管理措置**:雇用契約・委託契約における秘密保持義務、年1回以上の教育
- **物理的安全管理措置**:入退室管理、盗難防止、機器・媒体の持出制限、廃棄
- **技術的安全管理措置**:アクセス制御、識別認証、不正アクセス防止、暗号化、ログ管理(doc_045参照)

### 第25条(漏えい等発生時の対応)
個人データの漏えい、滅失、毀損又はそのおそれを認識した場合、直ちにCPO及びCSIRTに報告する。個人情報保護法施行規則第7条に定める漏えい等事案(要配慮個人情報、財産的被害のおそれ、不正アクセス、1,000人超のいずれかを含むもの)に該当する場合、以下を実施する。
1. 個人情報保護委員会への速報:概ね3〜5日以内
2. 個人情報保護委員会への確報:30日以内(不正アクセス起因は60日以内)
3. 本人への通知:事態把握後速やかに、状況に応じた適切な方法で
4. GDPR適用案件の場合、認識から72時間以内に監督機関へ通知(GDPR第33条)、本人通知は高リスクの場合に実施(同第34条)

詳細手順はdoc_031「インシデント対応手順書」及び別途定める「個人情報漏えい等対応マニュアル」による。

### 第26条(教育)
当社は、全役職員に対し年1回以上の個人情報保護教育を実施する。新入社員、新任管理職、個人データを大量に取り扱う部署の従業員には別途専門教育を行う。受講記録はLMSで管理し、未受講者には所属長から受講を促す。

### 第27条(違反時の措置)
本規程に違反した役職員には、就業規則及び懲戒規程に基づく処分を行うことがある。違反により個人情報保護法その他の法令違反となった場合は、所管官庁への報告、課徴金、刑事責任等を伴うことがある点を全役職員は認識すること。
