## 1. 研修の全体像
<!-- section_id: sec_1 -->

株式会社サンプルでは、新入社員(新卒・中途を問わず)の入社から3ヶ月間を「オンボーディング期間」と定義し、人事部・配属先・メンターが三位一体で立ち上がりを支援します。本章では、入社初日から本配属までのおおまかな流れを示します。

### 1.1 オンボーディング全体スケジュール

| 期間 | フェーズ | 主担当 | 主な内容 | 形式 |
| :--- | :--- | :--- | :--- | :--- |
| Day1 | 入社手続 | 人事部 | 入社式、雇用契約・誓約書、健康保険等の手続き、PC受領 | 集合 |
| Day2〜Day5 | 集合研修(座学) | 人事部・各部門講師 | 企業理念、就業規則、コンプライアンス、情報セキュリティ、ビジネスマナー | 集合 |
| Week2 | 部門横断研修 | 各部門代表 | 事業説明、部門ローテーション、業務システム概要 | 集合+見学 |
| Week3〜Week4 | OJT準備期 | 配属先OJT担当 | 業務マニュアル読込、シャドーイング、簡易タスク | OJT |
| Month2 | 実務OJT | 配属先上長・メンター | 本人タスク担当、週次1on1、課題提出 | OJT |
| Month3 | フォロー研修 | 人事部 | 振り返り研修、配属確定面談、本配属辞令 | 集合+面談 |

### 1.2 達成すべきマイルストーン

1. **入社1週間時点**: 社内システム(Google Workspace、Slack、KING OF TIME、Notion、Box)へのログインと基本操作が単独で可能。
2. **入社1ヶ月時点**: 配属部門の業務フローを説明でき、定型業務を補助付きで遂行可能。
3. **入社3ヶ月時点**: 定型業務を独力で遂行可能。OJT課題を完了し、本配属判定基準を満たす。

> **注意:** 集合研修期間中の遅刻・欠席は人事評価に影響します。やむを得ず欠席する場合は、当日8:45までに人事部・研修担当(内線2101)およびメンターへSlackで連絡してください。

### 1.3 関係部署と問合せ先

| 内容 | 担当 | 連絡先 |
| :--- | :--- | :--- |
| 研修全般・カリキュラム変更 | 人事部 人材開発課 | hr-dev@sample.co.jp / 内線2101 |
| PC・アカウント関連 | IT部 ヘルプデスク | it-help@sample.co.jp / 内線2201 |
| 経費・通勤費 | 経理部 | keiri@sample.co.jp / 内線2301 |
| メンタル不調・相談 | 人事部 産業保健担当 | health@sample.co.jp(秘匿対応) |

> **FAQ:** Q. 研修期間中に有給休暇は取得できますか?  A. 原則、集合研修期間(Day1〜Week2)中の有給取得はご遠慮ください。私傷病等やむを得ない場合は人事部に申し出てください。

## 2. カリキュラム
<!-- section_id: sec_2 -->

カリキュラムは「座学」「OJT」「eラーニング(LMS:Schoo for Business)」の3形式で構成されます。受講状況はLMSダッシュボードで自動集計され、人事部および配属先上長が閲覧します。

### 2.1 座学プログラム一覧

| No. | 科目名 | 講師 | 時間 | 実施日 | 必須/任意 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| S01 | 企業理念・中期経営計画 | 経営企画部長 | 90分 | Day2 AM | 必須 |
| S02 | 就業規則・人事制度 | 人事部長 | 120分 | Day2 PM | 必須 |
| S03 | ビジネスマナー基礎 | 外部講師 | 180分 | Day3 終日 | 必須 |
| S04 | 情報セキュリティ | IT部 セキュリティ課 | 90分 | Day4 AM | 必須 |
| S05 | 個人情報保護・GDPR | 法務部 | 90分 | Day4 AM | 必須 |
| S06 | コンプライアンス・贈収賄防止 | 内部監査室 | 90分 | Day4 PM | 必須 |
| S07 | ハラスメント防止 | 外部講師 | 90分 | Day4 PM | 必須 |
| S08 | 財務基礎(BS/PL/CF) | 経理部 | 120分 | Day5 AM | 必須 |
| S09 | 自社製品・サービス概要 | 各事業部 | 180分 | Day5 PM | 必須 |
| S10 | プレゼンテーション基礎 | 外部講師 | 180分 | Week2 | 任意 |

### 2.2 eラーニング必修科目(LMS)

| コードコード | 講座名 | 標準学習時間 | 受講期限 |
| :--- | :--- | :--- | :--- |
| E-101 | ビジネス文書・敬語 | 2.0h | 入社1ヶ月以内 |
| E-102 | Google Workspace操作(Gmail/Drive/Meet) | 1.5h | 入社2週間以内 |
| E-103 | Slackコミュニケーション規程 | 1.0h | 入社1週間以内 |
| E-104 | 情報セキュリティ年次研修 | 2.0h | 入社1ヶ月以内 |
| E-105 | 反社会的勢力排除研修 | 1.0h | 入社1ヶ月以内 |

### 2.3 OJTカリキュラム標準フォーマット

1. 配属初日にOJT担当者と「OJT計画書(Notion テンプレ:OJT-PLAN-01)」を共同作成。
2. 計画書には①習得スキル、②達成基準、③チェックポイント日(2週・1ヶ月・2ヶ月)を必ず明記。
3. 毎週金曜17:00までに新入社員が「週次振り返り(KPT)」をNotionに記載、OJT担当がコメント。
4. 1ヶ月ごとに人事部・OJT担当・本人で三者面談を実施。

> **NG例:** OJT計画書を作成せず「とりあえずやってみて」と指示する → 達成基準が曖昧で評価不能。
>
> **OK例:** OJT計画書に「2週間後までにSalesforceで案件登録を10件単独実施できる」のように具体的・計測可能な基準を記載する。

> **注意:** eラーニング未受講のまま月次締めを迎えると、配属先上長宛にアラートメールが自動送信されます。

## 3. メンター制度
<!-- section_id: sec_3 -->

メンター制度は新入社員の早期戦力化と心理的安全性確保を目的としています。OJT担当(業務指導)とは別に、入社3〜5年目の先輩社員をメンターとして指名し、業務以外の悩みや中長期キャリア相談にも対応します。

### 3.1 メンターの役割と禁止事項

| 役割 | 内容 |
| :--- | :--- |
| 業務相談の窓口 | OJT担当に聞きづらい質問の一次受け |
| キャリア相談 | 中長期のキャリアパス、社内異動・職種転換 |
| 心理的サポート | 環境変化への適応支援、孤立防止 |
| 社内ネットワーク紹介 | 他部署キーパーソンへの引き合わせ |

> **注意:** メンターは人事評価権限を持ちません。1on1で得た情報を本人の同意なく上長・人事部へ共有することは禁止です(ハラスメント・健康上の重大リスクを除く)。

### 3.2 実施頻度とフォーマット

1. **頻度**: 入社後3ヶ月間は週1回30分、4〜6ヶ月目は隔週30分、7〜12ヶ月目は月1回60分。
2. **形式**: 原則1on1(対面またはGoogle Meet)。社外カフェでの実施可(費用は1回1名2,000円まで会社負担)。
3. **記録**: Notion「メンターログ(MENT-LOG-01)」テンプレに、議題・本人の状態・次回アクションを記録。本人と共有可、人事部閲覧不可。

### 3.3 1on1標準アジェンダ(テンプレ)

| 時間 | 内容 |
| :--- | :--- |
| 0:00〜0:05 | アイスブレイク、体調・睡眠の確認 |
| 0:05〜0:15 | 今週のWin/Challenge共有 |
| 0:15〜0:25 | 困りごと・相談事項のディスカッション |
| 0:25〜0:30 | 次週アクション、次回日程確定 |

### 3.4 メンター手当と表彰

メンターには月額5,000円(課税)を給与で支給します。半期ごとに人事部が新入社員アンケートを実施し、上位5名を「Best Mentor Award(賞金30,000円)」として全社表彰します。

> **FAQ:** Q. メンターと相性が合わない場合は?  A. 入社2ヶ月時点までは人事部に申し出ることで無条件交代可能です。気兼ねなく hr-dev@sample.co.jp までご連絡ください。

## 4. 評価とフィードバック
<!-- section_id: sec_4 -->

研修期間中の評価は、最終的な本配属可否および初任配属後の処遇に直結します。評価は「中間評価(1ヶ月時点)」「最終評価(3ヶ月時点)」の2回実施します。

### 4.1 評価軸と配点

| 評価項目 | 評価者 | 配点 | 主な観点 |
| :--- | :--- | :--- | :--- |
| 業務遂行力 | OJT担当・上長 | 30点 | 指示理解、正確性、納期遵守 |
| 学習意欲 | 人事部・メンター | 20点 | eラーニング受講率、自主学習、質問力 |
| 成長度 | 上長・メンター | 20点 | 入社時からの変化量、改善サイクル |
| 協調性 | 部門メンバー(360度) | 15点 | 報連相、チームへの貢献 |
| コンプライアンス | 上長・人事部 | 15点 | 規程遵守、セキュリティ事故有無 |
| **合計** | - | **100点** | - |

### 4.2 評価シート構成(中間評価シート HR-EVAL-MID-01)

1. 基本情報(氏名、配属、評価期間、評価者)
2. 評価項目ごとの5段階評定(1=要改善 〜 5=期待超え)とコメント
3. 本人自己評価欄(同一項目を本人が自己採点)
4. ギャップ分析(評価者と本人の差分2点以上の項目を抽出)
5. 次月アクションプラン(本人記入、上長承認)

### 4.3 フィードバック面談の進め方

1. 面談時間は60分を確保し、会議室予約必須(Outlook で予約)。
2. 冒頭で本人の自己評価を聞く(評価者が先に話さない)。
3. 強み・改善点を**事実ベースで具体例を示す**(SBI法:Situation-Behavior-Impact)。
4. 改善点は必ず**期限とアクション**をセットで合意。
5. 面談録は人事システム(SmartHR)に7日以内に登録。

> **NG例:** 「コミュニケーションが弱い」と抽象的に伝える → 本人が何を直せばよいか分からない。
>
> **OK例:** 「先週の定例で議事録更新が金曜まで遅れた事例(S)、要因として確認質問が少なかった点(B)、結果として2名が翌週の作業着手を遅らせた(I)」と具体化する。

### 4.4 評価結果と本配属判定

| 合計点 | 判定 | 取扱い |
| :--- | :--- | :--- |
| 80点以上 | A判定 | 第一希望配属を優先検討 |
| 60〜79点 | B判定 | 適性と部門需要を踏まえ配属 |
| 40〜59点 | C判定 | 配属保留、追加研修1ヶ月 |
| 40点未満 | D判定 | 人事部・本人で再面談、配置再検討 |

> **注意:** 評価結果に異議がある場合、結果通知から5営業日以内に「評価申立書」を人事部長宛に提出してください。再評価委員会(人事部長、配属先部門長、社外取締役1名)で審議します。

> **FAQ:** Q. C判定だった場合、自動的に解雇となりますか?  A. いいえ。追加研修と再評価の機会を設けます。試用期間満了時の処遇は別途、就業規則第15条に基づき判断されます。
