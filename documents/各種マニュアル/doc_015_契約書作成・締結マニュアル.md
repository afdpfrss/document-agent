## 1. 契約書テンプレート
<!-- section_id: sec_1 -->

株式会社サンプルでは、契約書作成のリードタイム短縮と品質統一のため、法務部が標準テンプレートを整備しています。テンプレートは **Box > Legal > Contract Templates** に集約され、全社員が閲覧可能です(編集は法務部のみ)。

### 1.1 標準テンプレート一覧

| テンプレID | 種別 | 対象取引 | 標準有効期間 | 保管パス |
| :--- | :--- | :--- | :--- | :--- |
| TMP-NDA-01 | 秘密保持契約(双方向) | 取引検討段階 | 3年 | Box/Legal/Contract Templates/NDA |
| TMP-NDA-02 | 秘密保持契約(片務:相手方が情報開示) | 受託検討 | 3年 | 同上 |
| TMP-OUT-01 | 業務委託契約(準委任) | コンサル・運用受託 | 1年(自動更新) | Box/Legal/Contract Templates/Outsourcing |
| TMP-OUT-02 | 業務委託契約(請負) | 開発・制作物受託 | 案件単位 | 同上 |
| TMP-SAL-01 | 売買基本契約 | 物品売買 | 1年(自動更新) | Box/Legal/Contract Templates/Sales |
| TMP-SAL-02 | 個別売買契約 | 個別注文 | 単発 | 同上 |
| TMP-SUB-01 | サブスクリプション契約 | SaaS提供 | 1年(自動更新) | Box/Legal/Contract Templates/Subscription |
| TMP-LIC-01 | ライセンス契約 | 知財ライセンス | 3年 | Box/Legal/Contract Templates/License |
| TMP-TOS-01 | 利用規約 | Webサービス利用者向け | 都度改定 | Box/Legal/Contract Templates/TOS |
| TMP-COL-01 | 共同研究開発契約 | 共同R&D | 案件単位 | Box/Legal/Contract Templates/Collaboration |
| TMP-AGT-01 | 代理店契約 | 販売代理 | 1年(自動更新) | Box/Legal/Contract Templates/Agency |
| TMP-EMP-01 | 雇用契約(正社員) | 採用 | 期間の定めなし | Box/HR(別管理) |

### 1.2 テンプレート利用フロー

1. Boxから該当テンプレートをダウンロード(Word .docx)。
2. ファイル名を `[契約種別]_[相手方略称]_[YYYYMMDD]_v[版数].docx` に変更(例:`NDA_株式会社A_20260519_v1.docx`)。
3. 黄色マーカー箇所(契約相手方名、契約金額、契約期間等)を埋める。
4. 標準条項を **修正する場合は赤字で明示**。修正理由をコメント機能で記載。
5. 軽微な修正(社名、日付等)のみであれば法務レビュー不要。

### 1.3 修正の度合いと対応

| 修正度合い | 例 | 法務レビュー |
| :--- | :--- | :--- |
| 軽微 | 社名・住所・金額・期間の数値変更のみ | 不要 |
| 中度 | 標準条項の一部変更、相手方修正案の受領 | 必要 |
| 重度 | 相手方フォーマットを使用、定型外の取引 | 必須 |

> **NG例:** 標準テンプレに大幅な修正を加えて法務レビューを経ずに締結 → 不利な条項が含まれていた場合、責任は起票部門に帰属。
>
> **OK例:** 修正度合いが「中度」を超えそうな場合は迷わず法務レビュー依頼を起票。

### 1.4 テンプレート改定

法令改正・判例変更・経営方針変更に応じて、法務部が年2回(4月・10月)にテンプレートを見直します。改定告知は社内ポータル(Notion)および社内メール(legal-news@)で行われます。改定済テンプレを使用する義務があり、旧版テンプレで起票すると差戻されます。

## 2. 法務レビュー依頼
<!-- section_id: sec_2 -->

法務レビューは、契約リスクを軽減し当社の権利・義務を適切に設計するための重要プロセスです。チケット管理は **ServiceNow > 法務サービス > 契約レビュー** で行います。

### 2.1 レビュー必須となる契約

以下のいずれかに該当する場合、法務レビューは必須です。

1. 契約金額が **税込500万円以上**
2. 契約期間が **1年超**(自動更新含む)
3. **非定型条項** を含む(損害賠償上限、知的財産権帰属、競業避止、独占的取引等)
4. **海外当事者** との契約
5. 当社が **相手方フォーマット** を受け入れる場合
6. **個人情報** または **機微情報** を扱う契約
7. **業務提携・資本提携・M&A** に関する契約

### 2.2 レビュー依頼の起票

1. ServiceNow ログイン → **[サービスカタログ] > [契約レビュー依頼]** をクリック。
2. 入力項目:

| 項目 | 必須 | 内容 |
| :--- | :--- | :--- |
| 案件名 | 必須 | 例:「A社向けSaaS提供契約」 |
| 取引概要(背景・目的) | 必須 | 200字以上 |
| 相手方(社名・所在地・業種) | 必須 | - |
| 契約金額(税抜) | 必須 | 例:1,200万円 |
| 契約期間 | 必須 | 例:2026/06/01〜2027/05/31、自動更新あり |
| 契約形態 | 必須 | 当社テンプレ/相手方テンプレ/共同作成 |
| 特殊条項の有無 | 必須 | 競業避止/独占/最恵国待遇/SLA等 |
| 緊急度 | 必須 | 通常/急ぎ/最優先 |
| 希望納期 | 必須 | YYYY/MM/DD |
| 添付ファイル | 必須 | ドラフト .docx、相手方提示資料 |

3. 上長承認(課長以上)後、法務部にチケット自動アサイン。

### 2.3 レビューSLA

| 緊急度 | 標準納期 | 条件 |
| :--- | :--- | :--- |
| 通常 | 5営業日 | 制限なし |
| 急ぎ | 3営業日 | 上長承認+理由明記 |
| 最優先 | 1営業日 | 部長承認+理由明記、月3件まで |

> **注意:** 「急ぎ・最優先」を多用すると法務部のキャパシティを圧迫し、他案件にも影響します。原則は「通常」、契約交渉の節目から逆算して余裕を持って依頼してください。

### 2.4 レビュー結果の受領

1. レビュー完了時、ServiceNowチケットに以下が添付されます。
   - 修正提案入りドラフト(Word変更履歴)
   - 法務コメント(リスク評価:高/中/低、交渉ポイント)
   - リスクサマリ(主要争点を箇条書き)
2. 内容を確認のうえ、相手方への提示またはさらに法務と協議。
3. 相手方からの修正案を受領した際は、再度同じチケットに添付し再レビュー依頼。

### 2.5 緊急時(休日・夜間)

休日・夜間の緊急法務相談は、以下のホットラインを利用してください(月間2件まで)。

- 緊急連絡先: 法務部長 直通(080-XXXX-XXXX)
- メール: `legal-urgent@sample.co.jp`
- 顧問弁護士(法務部経由): 〇〇法律事務所

> **FAQ:** Q. 相手方から「サインしたら戻して」と契約書が送られてきました。レビュー不要ですか?  A. 不要ではありません。相手方フォーマットは必ずレビュー必須です。ServiceNow起票してください。

## 3. 押印手続き
<!-- section_id: sec_3 -->

押印は会社としての意思表示の最終確認です。種類と権限を理解し、適切に運用してください。当社は **「印章管理規程」(社内ポータル参照)** に基づき押印を実施します。

### 3.1 印章の種類

| 印章 | 用途 | 管理者 | 押印権限者 |
| :--- | :--- | :--- | :--- |
| 代表印(実印) | 重要契約、官公庁提出書類 | 法務部長 | 代表取締役 |
| 銀行印 | 金融機関手続き | 経理部長 | CFO |
| 社印(角印) | 請求書、見積書、社内文書 | 総務部 | 部長以上 |
| 部門印 | 部門内回付・受領印 | 各部門 | 課長以上 |
| 認印 | 軽微な確認 | 各個人 | 本人 |

### 3.2 電子契約と紙契約の使い分け

| 取引種別 | 推奨形式 | 備考 |
| :--- | :--- | :--- |
| 国内一般契約 | **電子契約(CloudSign)** | 印紙税不要、保管も電子 |
| 海外取引 | **電子契約(DocuSign)** | 多言語対応 |
| 官公庁・金融機関提出 | 紙(実印・印鑑証明) | 制度上電子不可のケースあり |
| 不動産登記関連 | 紙(実印) | 法令上原本必要 |
| 既存顧客の更新契約 | 既存締結方式を継承 | 顧客都合に合わせる |

### 3.3 紙契約の押印申請手順

1. ServiceNow > **[サービスカタログ] > [押印申請]** から起票。
2. 入力項目:

| 項目 | 必須 | 例 |
| :--- | :--- | :--- |
| 文書名 | 必須 | A社業務委託契約書 |
| 契約相手方 | 必須 | 株式会社A |
| 押印種別 | 必須 | 代表印/社印/銀行印 |
| 押印枚数 | 必須 | 2部(双方保管) |
| 法務レビュー番号 | 該当時必須 | LEG-2026-0042 |
| 添付 | 必須 | 押印対象PDF |

3. 法務部または総務部にて承認 → 申請者が押印場所(法務部:本社5F、総務部:本社2F)で受領・押印。
4. 契約相手方への送付後、双方押印済原本を **契約原本書庫(本社5F)** に保管(法務部管理)。
5. PDF化したコピーを **Box > Legal > Signed Contracts** に保存。

### 3.4 電子契約(CloudSign)の手順

1. CloudSign(sample.cloudsign.jp)にSSOログイン。
2. **[新規書類アップロード]** から契約書PDFを選択。
3. 署名者の入力欄(社名・氏名・メール)、押印位置をドラッグ操作で配置。
4. 当社代表者(または受任者)へ社内承認ルートを設定。
5. 相手方の署名後、自動で電子署名+タイムスタンプ付与済PDFが完成。
6. 完成PDFは自動的にBox連携で **Box > Legal > Signed Contracts** に保管。

### 3.5 印章の貸出・持出禁止

1. 代表印・銀行印は **持出厳禁**。法務部・経理部内での押印のみ。
2. 社印・部門印を会議室等へ持ち出す場合は、**当日中の返却** を厳守。
3. 印章を私物化・複製することは絶対禁止(私文書偽造罪に該当する可能性)。

> **注意:** 押印は「会社の意思表示」です。本人が内容を理解せず押印するのは絶対に避けてください。「押すだけだから」と頼まれても、内容確認・法務レビュー完了を確認するまで押印してはいけません。

## 4. 契約管理
<!-- section_id: sec_4 -->

締結済契約は **契約ライフサイクル管理(CLM)** の対象として、組織的に管理します。当社のCLM基盤は **CloudSign(電子契約)+契約台帳(Notion DB)** の組合せです。

### 4.1 契約台帳への登録

紙・電子を問わず、すべての締結契約は契約台帳に登録します。

| 項目 | 必須 | 例 |
| :--- | :--- | :--- |
| 契約ID | 自動採番 | CTR-2026-0185 |
| 契約名 | 必須 | A社業務委託契約 |
| 相手方 | 必須 | 株式会社A |
| 契約種別 | 必須 | 業務委託(準委任) |
| 締結日 | 必須 | 2026/05/19 |
| 契約開始日 | 必須 | 2026/06/01 |
| 契約終了日 | 必須 | 2027/05/31 |
| 自動更新 | 必須 | あり(1年) |
| 解約予告期間 | 必須 | 終了3ヶ月前まで |
| 契約金額(年額) | 必須 | 1,200万円 |
| 主管部門 | 必須 | 営業1課 |
| 主担当 | 必須 | 田中太郎 |
| 法務レビュー番号 | 該当時 | LEG-2026-0042 |
| 原本保管場所 | 必須 | Box/紙原本書庫 |

### 4.2 更新・解約アラート

Notion 契約台帳には、以下のアラートが自動設定されています。

| アラート | 通知タイミング | 通知先 |
| :--- | :--- | :--- |
| 更新6ヶ月前 | 6ヶ月前の月初 | 主担当+上長 |
| 更新3ヶ月前 | 3ヶ月前の月初 | 主担当+上長+法務部 |
| 更新1ヶ月前 | 1ヶ月前の月初 | 主担当+上長+法務部+部長 |
| 解約予告期限7日前 | 期限7日前 | 主担当+部長 |

更新意思確認は主担当の責任で行い、変更がある場合は **契約改定** または **解約通知** を期限内に発出します。

### 4.3 契約改定の手順

1. 改定が必要となった時点で法務レビュー依頼を起票。
2. 改定協議(条項追加・削除・修正)。
3. **覚書** または **改定契約書** を作成し、両当事者が押印・電子署名。
4. 契約台帳にバージョン管理(v2、v3 …)。

### 4.4 解約手続き

1. 解約予告期間に注意し、**書面(電子可)** にて相手方へ解約通知。
2. 解約通知書テンプレは Box > Legal > Template > 解約通知 を使用。
3. 通知発出後、契約台帳のステータスを **「解約予定」** に更新。
4. 契約終了日経過後 **「終了」** に更新。
5. 解約後の機密情報破棄・返還処理を相手方と協議。

### 4.5 監査・保管

1. 契約原本(紙)は **締結から契約終了+10年間** 保管(税務調査対応)。
2. 電子契約は CloudSign + Box にて永年保管。
3. 内部監査(年1回)で契約台帳と原本の突合実施。
4. 個人情報を含む契約は **個人情報保護台帳** にも別途登録。

> **注意:** 契約原本の紛失は重大インシデントです。万一紛失した場合は速やかに法務部長に報告し、相手方への再締結交渉、内部監査・必要に応じて当局報告等の対応を行います。

> **FAQ:** Q. 契約台帳のNotion更新が面倒です。  A. CloudSign締結時はWebhook連携で自動登録されます。手動入力は紙契約のみ。詳細はIT部にお問合せください。
