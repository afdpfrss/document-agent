## 1. 出退勤打刻
<!-- section_id: sec_1 -->

株式会社サンプルでは、勤怠管理システムとして **KING OF TIME(キングオブタイム)** を全社共通で導入しています。打刻方法はPC・スマートフォン・ICカードの3種類があり、勤務形態(オフィス出社・在宅勤務・外勤)に応じて使い分けます。

### 1.1 ログイン方法

1. PCブラウザで `https://s2.kingtime.jp/admin` にアクセス。
2. **企業ID:`sample-corp`**、**ログインID:社員番号(s00000)**、初期パスワードでログイン。
3. 初回ログイン後、パスワード変更とMFA(Authenticatorアプリ)設定必須。
4. SSO連携済のため、Google Workspaceアカウントでもログイン可。

### 1.2 打刻方法一覧

| 方法 | 利用デバイス | 利用ケース | 認証方式 |
| :--- | :--- | :--- | :--- |
| Webタイムレコーダー | PCブラウザ | オフィス出社・在宅勤務 | SSO+IPアドレス制限 |
| スマホアプリ | iOS/Android | 直行直帰・外勤 | 顔認証+GPS |
| ICカード打刻機 | 本社1F・各支社受付 | 出社時の物理打刻 | 社員証(FeliCa) |
| Slack連携(/punch) | Slack | 在宅勤務の簡易打刻 | SSO |

### 1.3 基本打刻(出社時)

1. PCを起動し、ブラウザのブックマーク「KING OF TIME」をクリック。
2. ログイン後、ダッシュボード上部の **[出勤]** ボタンをクリック。
3. 「打刻が完了しました」と表示されれば正常完了。
4. 退勤時は同様に **[退勤]** ボタンをクリック。

### 1.4 休憩打刻

1. 昼休憩等で離席する際 **[休憩開始]** をクリック。
2. 復帰時 **[休憩終了]** をクリック。
3. 法定休憩(労働時間6時間超で45分、8時間超で60分)は **自動的には付与されません**。各自必ず休憩打刻を実施してください。

### 1.5 外出・戻り

1. 業務上の外出時(取引先訪問等)は **[外出]** をクリック → 戻り次第 **[戻り]** をクリック。
2. 外出時間は労働時間としてカウント(直行直帰の場合は **[出勤]/[退勤]** で代用)。

### 1.6 標準勤務時間

| 項目 | 内容 |
| :--- | :--- |
| 所定労働時間 | 7時間45分(9:00〜17:45、休憩1時間) |
| コアタイム(フレックス) | 10:00〜15:00 |
| フレックス利用可能時間 | 7:00〜22:00 |
| 在宅勤務時の標準勤務 | 同上(打刻必須) |

### 1.7 打刻忘れの修正

1. KING OF TIMEログイン → **[マイ勤務] > [打刻申請]** を選択。
2. 該当日の打刻箇所をクリック → 正しい時刻を入力。
3. 修正理由を50字以上で入力(例:「在宅勤務開始時にPC起動忘れ、9:00から業務開始」)。
4. **[申請]** → 上長承認後、勤怠データに反映。
5. 申請は **当日中、遅くとも翌営業日中** に行うこと。

> **NG例:** 月末にまとめて打刻修正申請 → 上長承認の負担増+月次締めに間に合わない。
>
> **OK例:** 打刻忘れに気づいた即時に申請、理由を具体的に記載。

### 1.8 不正打刻防止

KING OF TIME には以下の不正検知機能があります。

1. **IPアドレス制限**: オフィスWi-Fiまたは会社VPN経由でないと出社打刻不可。
2. **GPS判定**: スマホ打刻時はGPS座標を記録、申告場所と乖離がある場合は要確認。
3. **顔認証**: スマホ打刻時にカメラで顔照合(初回登録必要)。
4. **代理打刻禁止**: 他者が打刻した場合、ログ追跡により発覚。懲戒対象。

## 2. 残業申請
<!-- section_id: sec_2 -->

時間外労働(残業)は労使協定(36協定)および労働基準法に基づき、**事前申請・上長承認** を原則とします。

### 2.1 残業申請の種別

| 種別 | 申請タイミング | 承認権限 |
| :--- | :--- | :--- |
| 事前申請 | 当日17:00まで | 直属上長(課長) |
| 事後申請 | 翌営業日中 | 直属上長(課長) |
| 深夜残業(22:00以降) | 事前申請のみ | 部長 |
| 休日労働(土日祝) | 事前申請のみ | 部長 |
| 月45時間超過 | 事前申請のみ | 部長+人事部 |

### 2.2 申請手順(事前)

1. KING OF TIME > **[マイ勤務] > [残業申請]** をクリック。
2. 以下の入力項目を埋める。

| 項目名 | 必須/任意 | 記入例 |
| :--- | :--- | :--- |
| 申請日 | 必須 | 2026/05/19 |
| 予定開始時刻 | 必須 | 17:45 |
| 予定終了時刻 | 必須 | 20:00 |
| 残業時間 | 自動計算 | 2時間15分 |
| 残業理由(80字以上) | 必須 | A社向け提案書を翌日10時提出のため、本日中の修正反映が必要 |
| 案件コード | 推奨 | PRJ-2026-042 |

3. **[申請]** → 上長Slack/メール通知 → 承認後に残業実施。

### 2.3 上限管理

36協定により、当社の時間外労働には以下の上限があります。

| 単位 | 原則 | 特別条項 |
| :--- | :--- | :--- |
| 月 | 45時間 | 80時間(年6回まで) |
| 年 | 360時間 | 720時間 |
| 複数月平均 | - | 80時間以内 |
| 連続休日 | 4週で4日以上 | - |

> **注意:** 月の累積残業が30時間を超えた段階で、KING OF TIMEから本人・上長・人事部にアラートメールが自動送信されます。45時間到達前に必ず業務調整を行ってください。

### 2.4 深夜残業

22:00〜翌5:00の労働は深夜割増(25%)の対象となります。深夜残業は健康影響が大きいため、原則として **当日中の業務完了は翌日に持ち越し** を推奨します。やむを得ない場合のみ部長承認のもと実施可。

### 2.5 在宅勤務時の残業

在宅勤務でも残業申請ルールは同じです。PCログ・Slackログ・KING OF TIMEログを突合し、申請残業時間との乖離があれば人事部から確認連絡があります。

> **NG例:** 申請せずに残業し、月末に「実は45時間超えていた」と発覚 → 36協定違反。会社・上長が労基署対応を要する。
>
> **OK例:** 事前申請を徹底し、想定外の業務発生時も都度申請。

## 3. 休暇申請
<!-- section_id: sec_3 -->

休暇は計画的に取得し、心身のリフレッシュと業務継続の両立を図ります。

### 3.1 休暇種別一覧

| 種別 | 単位 | 申請期限 | 給与 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| 年次有給休暇 | 1日/半日/時間(1時間刻み) | 原則3営業日前 | 全額支給 | 年5日取得義務 |
| 特別休暇(慶弔) | 1〜7日 | 事由発生後速やか | 全額支給 | 結婚・忌引・出産 |
| 振替休日 | 1日 | 休日労働前 | 全額支給 | 休日労働と相殺 |
| 代休 | 1日 | 休日労働後1ヶ月以内 | 通常分支給(割増は別途) | 休日労働後の代替休 |
| 産前産後休暇 | 規程による | 速やか | 規程による | 健保より給付 |
| 育児休業 | 規程による | 1ヶ月前 | 雇用保険給付 | 別途規程参照 |
| 介護休業 | 規程による | 2週間前 | 雇用保険給付 | 別途規程参照 |
| 病気休暇(無給) | 都度 | 事後可 | 無給 | 診断書提出 |

### 3.2 有給休暇の申請手順

1. KING OF TIME > **[マイ勤務] > [休暇申請]** をクリック。
2. 休暇種別「年次有給休暇」を選択。
3. 取得日・取得単位(全日/午前半休/午後半休/時間単位)を入力。
4. 業務引継内容を **[備考]** に記載(例:「A社対応は山田に依頼済、緊急時連絡可能」)。
5. **[申請]** → 上長承認。
6. 承認後、Google Calendarに自動で「休暇中」予定が登録される(任意で連携設定)。

### 3.3 申請期限

| 取得タイミング | 申請期限 |
| :--- | :--- |
| 通常の有給 | 3営業日前まで |
| 連続3日以上の有給 | 2週間前まで |
| 当日急病 | 始業時刻までに上長Slack連絡、後日有給/病気休暇として申請 |
| 慶弔休暇 | 事由発生後速やか(忌引きは事後申請可) |

### 3.4 半休・時間単位有給

1. **半休**: 午前(9:00-12:45)または午後(12:45-17:45)単位。
2. **時間単位有給**: 1時間刻み、年5日(40時間)分まで利用可。
3. 通院・子の学校行事等のスポット利用に推奨。

### 3.5 振替休日と代休の違い

| 観点 | 振替休日 | 代休 |
| :--- | :--- | :--- |
| 申請タイミング | 休日労働の **事前** | 休日労働の **事後** |
| 割増賃金 | 不要(休日と平日の入替) | 必要(35%割増) |
| 取得期限 | 振替先日まで | 休日労働日から1ヶ月以内 |
| おすすめ | 計画的な休日労働 | 緊急発生時 |

> **注意:** 法定休日(日曜)の労働は割増率35%、所定休日(土曜)は25%です。代休取得しても割増賃金は別途支給されます。

### 3.6 有給取得義務(年5日)

労基法により、年10日以上付与される労働者は年5日の有給取得が義務化されています。当社では各四半期末に未取得者へリマインドが送られ、年内に取得できる計画を上長と立案します。

> **FAQ:** Q. 有給を上長から「忙しい時期だから取らないで」と言われました。  A. 有給取得は労働者の権利です。時季変更権(事業の正常な運営を妨げる場合に時期変更を求める権利)はありますが、拒否権はありません。人事部(jinji@sample.co.jp)へ匿名相談可能です。

## 4. 月次締め
<!-- section_id: sec_4 -->

勤怠の月次締めは **当月15日締め** で運用しています(給与計算サイクルとの整合のため)。

### 4.1 月次スケジュール

| 日付 | 作業 | 担当 |
| :--- | :--- | :--- |
| 当月16日 | 前月16日〜当月15日分の勤怠が暫定確定 | システム自動 |
| 16日〜20日 | 社員本人が勤怠確認・修正申請 | 社員 |
| 20日 | 本人承認締切 | 社員 |
| 21日〜22日 | 上長承認 | 上長 |
| 23日〜24日 | 人事部最終確認 | 人事部 |
| 25日 | 給与計算 | 経理部 |
| 翌月5日 | 給与振込 | 経理部 |

### 4.2 本人確認手順

1. 毎月16日にKING OF TIMEから「勤怠確認のお願い」メール受信。
2. ログイン後 **[マイ勤務] > [勤怠確認]** を開く。
3. 当月の打刻一覧、残業時間、休暇取得状況を確認。
4. 不整合があれば該当日の **[修正申請]** から修正。
5. 問題なければ **[本人確認]** ボタンをクリック。

### 4.3 上長確認・承認

1. 上長のダッシュボードに「未承認の勤怠 ◯件」が表示。
2. 部下一覧から各人の勤怠を確認。
3. 残業上限超過・有給未取得・打刻漏れがあれば本人に差戻し。
4. 全件確認後、一括 **[承認]** をクリック。

### 4.4 修正依頼(締め後)

月次締め後の修正は原則受付けませんが、以下の例外時のみ人事部対応:

1. システム不具合による打刻記録欠損(IT部証明)
2. 病欠・事故等で本人確認不能だった場合(診断書等添付)
3. 上長の操作ミスによる承認漏れ

修正依頼は **`jinji-kintai@sample.co.jp`** 宛にメール、件名「【勤怠修正依頼】社員番号-氏名」で送付してください。

### 4.5 エラー時の対処

| エラー | 原因 | 対処 |
| :--- | :--- | :--- |
| 「打刻不整合」 | 出勤打刻のみで退勤打刻なし等 | 修正申請から補完 |
| 「休憩不足」 | 6h超勤務で休憩45分未満 | 休憩打刻を追加申請 |
| 「残業未申請」 | 退勤時刻が定時+30分以上で残業申請なし | 事後残業申請を起票 |
| 「有給日数不整合」 | 取得日数と残日数が合わない | 人事部へ確認依頼 |

> **注意:** 本人確認・上長承認の遅延は給与計算に直結します。期限内対応を徹底してください。3ヶ月連続で締めに遅延した場合、人事評価コメントの対象となります。

> **FAQ:** Q. 月途中で異動になりました。月次締めはどうなりますか?  A. 異動日を境に旧所属上長と新所属上長の双方が承認します。本人確認は通常通り20日までに実施してください。
