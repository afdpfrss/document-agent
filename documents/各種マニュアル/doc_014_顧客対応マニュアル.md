## 1. 電話応対
<!-- section_id: sec_1 -->

電話は会社の第一印象を決定する重要な接点です。株式会社サンプルの社員は、社内・社外を問わず、明るく丁寧な応対を心がけます。

### 1.1 基本ルール

| 項目 | ルール |
| :--- | :--- |
| 着信応答 | 3コール以内に出る(4コール以降は「お待たせしました」を冒頭に追加) |
| 第一声 | 「お電話ありがとうございます、株式会社サンプル ◯◯部の△△でございます」 |
| 声のトーン | 普段より1段明るく、口角を上げる |
| 話す速度 | 通常会話より20%程度ゆっくり |
| 復唱 | 会社名・氏名・電話番号・要件を必ず復唱 |
| 保留 | 30秒以内、超える場合は一度応答して状況説明 |

### 1.2 シーン別スクリプト集

**(1) 取次ぎ**
> 「◯◯の件でございますね。担当の△△におつなぎいたします。少々お待ちくださいませ。」
> (保留 → 担当者へ「□□株式会社の◯◯様より△△の件でお電話です」)

**(2) 担当者不在(在席なし)**
> 「申し訳ございません。△△はあいにく外出しております。本日17時頃に戻る予定でございます。戻り次第こちらから折り返しお電話を差し上げましょうか?」
> (相手氏名・会社名・電話番号・要件をメモ)

**(3) 担当者不在(他電話中)**
> 「申し訳ございません。△△はただいま別の電話に対応しております。終わり次第折り返しお電話を差し上げますが、いかがでしょうか?」

**(4) 担当者不在(休暇中)**
> 「申し訳ございません。△△は本日休暇を取得しております。明日◯時に出社予定でございます。お急ぎでしたら、代わりに□□が承りますが、いかがでしょうか?」

**(5) 担当者不明・社名のみ**
> 「ご用件をお伺いし、適切な担当者へお取次ぎいたします。差し支えなければ、ご用件をお聞かせいただけますでしょうか?」

**(6) 営業電話**
> 「申し訳ございません。当社では新規のお取引のご案内はお断りしております。ご了承くださいませ。」
> (繰り返し電話の場合は会社名と担当者名を控え、上長報告)

### 1.3 取次ぎフロー

1. 用件と担当者を確認。
2. 「少々お待ちください」と告げて **保留ボタン** を押す。
3. 担当者を内線で呼び出し、相手情報を伝達。
4. 担当者が応答可能なら転送、不可能なら保留解除して状況説明。
5. 折り返し約束時は「ご連絡先」「希望時間帯」を必ず聞く。

### 1.4 伝言メモテンプレート

```
【電話伝言メモ】
受信日時: ◯月◯日 ◯時◯分
受信者: (受けた人の名前)
発信元会社: 
発信者氏名: 
電話番号: 
用件: 
折返要否: 要 / 不要
折返希望時間: 
緊急度: 高 / 中 / 低
```

### 1.5 よくあるNG/OK

> **NG例:** 「△△は今いません。後でかけ直してください」 → 不在理由・戻り時刻不明、相手の手間を考えていない。
>
> **OK例:** 「△△はあいにく外出中で、◯時頃戻る予定です。折り返しお電話差し上げますので、ご連絡先をお伺いできますでしょうか?」

> **注意:** 自分の名前を名乗らずに応答するのは厳禁です。第一声で必ず会社名と氏名を伝えてください。

## 2. メール応対
<!-- section_id: sec_2 -->

メールは記録に残るコミュニケーション手段です。誤送信・情報漏えい・SLA違反を防ぐため、以下のルールを順守してください。

### 2.1 返信SLA(サービスレベル合意)

| 受信時刻 | 1次返信期限 | 本格回答期限 |
| :--- | :--- | :--- |
| 平日9:00〜17:00 | 1営業時間以内 | 1営業日以内 |
| 平日17:00以降 | 翌営業日10:00まで | 翌営業日中 |
| 休日 | 翌営業日10:00まで | 翌営業日中 |

1次返信は「受領しました、◯日までに回答します」でも可。完全な回答を待たせるよりも、受領通知を早く出すことが重要です。

### 2.2 件名のルール

| パターン | 形式 | 例 |
| :--- | :--- | :--- |
| 通常 | 【用件種別】具体的な件名 | 【ご相談】次回MTG日程について |
| 緊急 | 【至急】具体的な件名 | 【至急】本日17時までの確認依頼 |
| 返信 | Re: を維持、不必要なRe:重複は削除 | Re: 【ご相談】次回MTG日程について |
| 添付付 | 【添付あり】を明示 | 【添付あり】4月度報告書 |

### 2.3 本文の基本構成

```
[会社名] [部署] [役職] [氏名] 様

いつもお世話になっております。
株式会社サンプルの[氏名]でございます。

[本題:1段落で要点]

[詳細・経緯]

[依頼事項・期限]

お忙しいところ恐縮ですが、ご確認のほどよろしくお願い申し上げます。

[署名]
```

### 2.4 Cc/Bcc使い分け

| 用途 | To | Cc | Bcc |
| :--- | :--- | :--- | :--- |
| アクション要求相手 | ○ | - | - |
| 情報共有のみ(返信不要) | - | ○ | - |
| 上長への報告 | - | ○ | - |
| 大量同報(各受信者を秘匿) | - | - | ○ |
| 外部メルマガ等 | - | - | ○ |

### 2.5 添付ファイルの取扱い

1. **暗号化**: 機密性の高いファイルは **Box共有リンク + 期限7日設定** に切替を推奨。
2. **PPAP(パスワード別送)は禁止**: セキュリティ向上ではなく形式的負担となるため、Box共有を必須化。
3. **ファイルサイズ**: 添付は10MB以内。超える場合はBox共有。
4. **ファイル名**: 「日付_案件名_版数.拡張子」(例:`20260519_A社提案書_v2.pdf`)。

### 2.6 誤送信防止

| 対策 | 内容 |
| :--- | :--- |
| 送信前確認 | 宛先・添付・件名・本文の4点を声に出して確認 |
| 送信遅延 | Gmail設定で送信取消可能時間を30秒に設定(全社既定) |
| 外部送信警告 | sample.co.jp以外宛は送信時に確認ダイアログ表示 |
| 大量添付警告 | 5名以上のToに添付ファイル付き送信時はダブル確認 |
| DLP検知 | マイナンバー・クレジットカード番号等は自動ブロック |

> **NG例:** 急いで Cc/Bcc を取り違えて顧客リストを全員にCc送信 → 重大情報漏えい、即時報告対象。
>
> **OK例:** 大量配信時は必ずBccを使用、送信前にToに自分のメールアドレスを入れて自分にもテスト送信。

### 2.7 署名テンプレ

PCマニュアル(doc_011)第3章を参照してください。

> **FAQ:** Q. 海外取引先への返信、英語署名はどう設定しますか?  A. Gmail署名は複数登録可能です。「Default(英語)」「JP」を作成し、宛先言語に応じて切替できます。

## 3. クレーム対応
<!-- section_id: sec_3 -->

クレームはお客様からの貴重なフィードバックです。誠実かつ迅速な対応により、信頼回復と再発防止につなげます。

### 3.1 初期対応5原則

| No. | 原則 | 具体的行動 |
| :--- | :--- | :--- |
| 1 | 傾聴 | お客様の話を最後まで遮らず聞く。相槌を打ち、メモを取る |
| 2 | 共感・謝罪 | ご不便・ご不快な思いをさせたことに対して心からの謝罪 |
| 3 | 事実確認 | いつ・どこで・何が起きたか、5W1Hを明確化 |
| 4 | 対応策提示 | その場で結論を急がず、調査時間と回答期限を明示 |
| 5 | 再発防止 | 原因究明と再発防止策を文書化、対応完了後にフィードバック |

### 3.2 トークの組立

**Step 1: 受け止め**
> 「この度はご不便をおかけし、誠に申し訳ございません。詳しくお聞かせいただけますでしょうか。」

**Step 2: 傾聴(5〜10分はとにかく聞く)**
> 相槌「はい」「さようでございますか」「おっしゃる通りでございます」を適切に。
> 反論・弁明・言い訳は厳禁。

**Step 3: 事実整理と復唱**
> 「念のため、内容を整理させてください。◯月◯日に△△が起き、□□となってしまった、ということでよろしいでしょうか。」

**Step 4: 回答期限の約束**
> 「社内で確認のうえ、本日17時までに改めてご連絡いたします。それまでお時間をいただけますでしょうか。」

**Step 5: 感謝**
> 「貴重なご指摘をいただきまして、誠にありがとうございます。今後の改善に活かしてまいります。」

### 3.3 エスカレーション基準

下記いずれかに該当する場合、ただちに上長・関連部門にエスカレーションしてください。

| 区分 | 基準 | エスカ先 |
| :--- | :--- | :--- |
| 軽微 | 製品の通常不具合、軽い言葉のトーン | 課長 |
| 中等 | 損害賠償要求の可能性、繰り返しクレーム | 部長 |
| 重大 | 人的被害(健康被害)、SNS拡散済 | 部長+広報部+法務部 |
| 最重大 | 訴訟示唆、メディア取材、行政機関照会 | 役員+法務部+広報部+情報セキュリティ |

### 3.4 重大クレーム判定表

| 観点 | スコア(1〜5) |
| :--- | :--- |
| 損害金額 | 100万円未満1〜100万円以上5 |
| 影響顧客数 | 1名1〜100名以上5 |
| メディア/SNS拡散 | なし1〜拡散済5 |
| 健康被害 | なし1〜重篤5 |
| 法的請求 | なし1〜訴訟予告5 |

**合計スコア15点以上で重大クレーム**として扱い、社内対策本部設置を検討します。

### 3.5 やってはいけないこと

1. その場での即答(損害賠償額の確約、責任の即時認諾)
2. 担当者一人で抱え込む(必ず上長報告)
3. お客様情報を社内SNS等で共有(個人情報保護違反)
4. 録音・録画の無断実施
5. 「規程ですから」「決まりですので」の機械的回答

> **注意:** お客様から「録音している」と告げられた場合は、こちらも録音記録を残してよいか確認します。録音を拒否されてもこちらは記録メモを残します(後の証跡として重要)。

## 4. 記録とエスカレーション
<!-- section_id: sec_4 -->

顧客対応のすべては Salesforce Service Cloud(以下「Salesforce」)に記録し、ナレッジを蓄積します。

### 4.1 ケース起票

1. Salesforceにログイン → **[ケース] > [新規ケース]**。
2. 入力項目:

| 項目 | 必須 | 例 |
| :--- | :--- | :--- |
| ケースID | 自動採番(CASE-2026-XXXXX) | - |
| 顧客名(取引先) | 必須 | 株式会社○○ |
| 連絡者氏名 | 必須 | △△ 様 |
| 連絡チャネル | 必須 | 電話/メール/Web/訪問 |
| カテゴリ | 必須 | 問合せ/クレーム/要望/障害報告 |
| 製品 | 必須 | サンプルプロダクト Pro |
| 優先度 | 必須 | 緊急/高/中/低 |
| 件名 | 必須 | 50字以内で要約 |
| 説明 | 必須 | 詳細経緯・対応履歴 |
| 対応期限 | 推奨 | YYYY/MM/DD |
| 担当者 | 必須 | 自分または引継先 |

3. **[保存]** → ケースIDが採番され、後続の対応はすべてケースに紐づけて記録。

### 4.2 対応履歴の記録

1. ケース詳細画面 > **[活動] > [新規ToDo/メール/通話ログ]** で都度記録。
2. 通話は **「通話ログ」** で日時・通話相手・話した内容を記載(5W1H)。
3. メールは Salesforce Outlook Plugin で自動連携(送受信メールが自動でケースに紐付く)。
4. 訪問は **「ToDo」** で訪問日時・出席者・議事を記録。

### 4.3 ケースクローズ

1. すべての対応が完了したら **[ステータス]** を **「解決済」** → **「クローズ」** に更新。
2. **[クローズ理由]** を選択(顧客満足/代替案承認/対応不可等)。
3. **[再発防止策]** を記載(任意だが推奨)。
4. クローズ後、顧客満足度アンケート(NPS)が自動送信される。

### 4.4 エスカレーションフロー

```
[担当者] 
   ↓ 重大クレーム発生
[課長] (15分以内に状況把握)
   ↓
[部長] (30分以内、対応方針判断)
   ↓ 必要に応じて
[法務部] (法的リスク評価、24時間以内に見解)
[広報部] (メディア対応準備)
[情報セキュリティ] (情報漏えい関連時)
   ↓
[役員報告] (経営判断必要時、24時間以内)
   ↓
[対策本部設置] (最重大時、人員配置)
```

### 4.5 ナレッジ化

頻出する問合せ・クレームは Salesforce Knowledge に記事として登録し、社内および顧客向けFAQに展開します。月次でCS部門が以下を実施:

1. ケース集計(カテゴリ別件数、平均解決時間、CSAT)
2. 頻出問合せのナレッジ化(月10件以上の同種問合せ)
3. 製品改善依頼の取りまとめ(プロダクト部門へ連携)

> **注意:** Salesforce上の顧客情報・対応履歴は機密情報です。私用デバイスへのコピー、社外への口外、SNS投稿は懲戒対象となります。

> **FAQ:** Q. メール返信を Salesforce に手動コピペするのは手間です。  A. Outlook/Gmail プラグイン(Salesforce Inbox)を導入すれば自動連携されます。IT部に申請してください(無償)。
