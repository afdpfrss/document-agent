## 1. 予約方法
<!-- section_id: sec_1 -->

株式会社サンプル本社・支社の会議室は、Google Calendar(主)および Outlook(副)から予約します。両者は双方向同期されており、どちらから予約しても全社員に空き状況が共有されます。

### 1.1 会議室一覧(本社)

| フロア | 室名 | 定員 | 設備 | 用途目安 |
| :--- | :--- | :--- | :--- | :--- |
| 2F | 富士 | 20名 | プロジェクタ、Zoom Room、ホワイトボード×2 | 全社会議 |
| 2F | 北岳 | 12名 | 65型モニタ、Zoom Room | 部門会議 |
| 3F | 槍 | 8名 | 55型モニタ、Zoom Room | チーム定例 |
| 3F | 穂高 | 6名 | 43型モニタ、Zoom Room | 1on1、面談 |
| 4F | 雲海 | 4名 | モニタ、簡易電話会議 | 集中作業、Web会議 |
| 4F | 朝霧 | 2名 | モニタのみ | 1on1専用 |
| 5F | エグゼクティブルーム | 10名 | 役員専用、要事前申請 | 顧客対応・役員会 |

支社の会議室一覧は社内ポータル(Notion > Workplace > Meeting Rooms)を参照してください。

### 1.2 Google Calendar での予約手順

1. Google Calendar(calendar.google.com)を開く。
2. 任意の日時をクリック → **[その他のオプション]** で詳細画面へ。
3. タイトル(例:「営業部 週次定例」)、参加者(メールアドレス)を入力。
4. **[会議室・リソースの追加]** をクリック。
5. フロア・定員でフィルタし、空いている会議室を選択。
6. Web会議併用なら **[Google Meetを追加]** をオン(任意でZoomリンクに差替可)。
7. **[保存]** をクリック → 参加者へ自動的に招待メール送信。

### 1.3 Outlookでの予約手順

1. Outlook > **[予定表]** タブを開く。
2. **[新しい会議]** をクリック。
3. **[出席者]** に参加者を入力。
4. **[場所]** をクリック → 「会議室ファインダー」が起動 → 拠点・人数でフィルタ。
5. 空き時間の候補から選択し **[送信]**。

### 1.4 定例予約(繰り返し)

1. 詳細画面の **[繰り返し]** を **[毎週月曜 10:00-11:00]** 等に設定。
2. 終了日は **[6ヶ月後]** までを推奨(長すぎる定例は会議室を占有するため)。
3. 半期ごとに見直し、不要なら必ず削除。

### 1.5 予約ルール

| 項目 | ルール |
| :--- | :--- |
| 最長予約期間 | 当日から60日先まで |
| 単一予約最長時間 | 4時間(超える場合は総務部承認) |
| 同時間帯の重複予約 | 不可(システム制御) |
| 役員専用ルーム | 役員秘書経由で予約 |
| 来客対応 | 受付通知のため前日17時までに「来客票」をServiceNowに登録 |

> **NG例:** 「とりあえず押さえておこう」と毎日2時間×5室を仮押さえ → 他チームの予約を阻害、総務部から警告対象。
>
> **OK例:** 確定した予定のみを予約し、不要となれば即時キャンセル。

## 2. 利用ルール
<!-- section_id: sec_2 -->

会議室は共有資源です。次の利用者が気持ちよく使えるよう、以下のルールを徹底してください。

### 2.1 時間厳守

1. 開始時刻ちょうどに開始(参加者を待つために部屋外で待機)。
2. **終了予定時刻の5分前** に議論を切り上げ、退室準備に入る。
3. 次の利用者が入室を希望した場合は速やかに退室。
4. 延長が必要な場合は、次の利用者がいないことをCalendarで確認し、Calendar側で時間延長申請。

### 2.2 原状回復チェックリスト(退室時)

| No. | 項目 | 確認 |
| :--- | :--- | :--- |
| 1 | ホワイトボードを消去 | □ |
| 2 | 机を初期配置に戻す(レイアウト図はドアに掲示) | □ |
| 3 | 椅子を机の下に収納 | □ |
| 4 | プロジェクタ・モニタの電源OFF | □ |
| 5 | ゴミ(空容器・配布紙)を持ち帰る | □ |
| 6 | 忘れ物確認 | □ |
| 7 | 照明・エアコンOFF(最終利用者の場合) | □ |

### 2.3 飲食ルール

| 種別 | 可否 | 備考 |
| :--- | :--- | :--- |
| 蓋付き飲料(ペットボトル・タンブラー) | 可 | こぼした場合は本人清掃 |
| 缶飲料・紙コップ | 不可(エグゼクティブルーム除く) | - |
| 軽食・スナック | 不可(原則) | 終日研修等で必要な場合は総務承認 |
| 弁当・本格的な食事 | 不可 | カフェスペースを使用 |
| 取引先来客時の飲料提供 | 可 | 受付に「お茶出し依頼」 |

### 2.4 備品利用

1. ホワイトボードマーカー・イレーザーは室内常備(在庫切れは総務部へ Slack #facility)。
2. プロジェクタ・HDMIケーブルは室内常設。持ち出し禁止。
3. ノートPC接続用アダプタ(USB-C/HDMI)は自席から持参。
4. 延長コードは室内に1本。複数必要時は総務部から借用。

### 2.5 セキュリティ

1. 退室時は機密書類・PCを必ず持ち帰る(置き忘れは情報漏えい事故扱い)。
2. 来客同伴時は来客を一人にしない。離席時は同行者を残すか、来客と共に退室。
3. ホワイトボードに機密情報(顧客名・金額・人事情報等)を書いた場合、退室時に**確実に消去** + 写真確認推奨。

> **注意:** 来客との会議で重要書類を会議室に置き忘れた場合、情報セキュリティインシデントとして報告義務があります。IT部セキュリティ課(security@sample.co.jp)へ即時報告してください。

## 3. Web会議システム
<!-- section_id: sec_3 -->

株式会社サンプルでは Zoom(主)と Microsoft Teams(副)を併用しています。社内会議は Zoom、Microsoft系の取引先や Teams招待が来る案件は Teams を使用します。

### 3.1 Zoom Rooms(会議室常設)接続手順

1. 会議室入口のタッチパネル(Logitech Tap)を起動。
2. ホーム画面 **[Join]** をタップ。
3. ミーティングID(11桁)とパスコードを入力 → **[Join]**。
4. 開始後、パネル右下 **[Share Content]** で画面共有(BYODはHDMIケーブル接続またはAirPlay)。
5. 終了時 **[End]** > **[End Meeting for All]** または **[Leave Meeting]**(他参加者がいる場合)。

### 3.2 自席PCからのZoom会議

1. Zoomアプリ起動 → **[新規ミーティング]** または **[参加]**。
2. SSO ログイン(初回のみ。社名「sample」を入力)。
3. マイク・カメラの動作確認(設定 > オーディオ・ビデオ)。
4. ホスト権限付与は **[参加者] > [...] > [ホストにする]**。

### 3.3 ハイブリッド会議のベストプラクティス

会議室+リモート参加者が混在する場合、以下の構成を推奨します。

| 役割 | 推奨機材 |
| :--- | :--- |
| 室内集音 | Zoom Room設置の天井マイク(複数話者対応) |
| 室内表示 | Zoom Roomの大型モニタ(リモート参加者の顔表示) |
| 室内発言者 | 中央スピーカーに向かって発言。輪番制での発言推奨 |
| ファシリテーター | チャット欄を常時モニタし、リモート発言を拾う |
| 画面共有 | 会議室PCまたは発表者個人PCから Zoom 経由で共有 |

> **NG例:** 室内参加者全員が個別ノートPCからZoom接続 → ハウリング、音声品質著しく低下。
>
> **OK例:** 室内は Zoom Room 1台に集約、個人PCはマイクOFF/スピーカーOFFで補助参加。

### 3.4 トラブル対処

| 症状 | 原因 | 対処 |
| :--- | :--- | :--- |
| 音声が聞こえない | Zoom音声未参加 | 「コンピューターオーディオに参加」をクリック |
| ハウリング | 室内で複数マイクON | 1台に集約 |
| 画面共有できない | 共有許可OFF | ホストに権限付与依頼 |
| Zoom Room が応答しない | パネル不具合 | 電源コンセント再起動(最後の手段)、Slack #it-helpdesk |

### 3.5 録画ルール

1. 録画は事前に全参加者の同意を得る(冒頭で口頭確認)。
2. クラウド録画のみ可(ローカル録画は禁止)。
3. 録画ファイルはZoomクラウドから自動でBoxへ転送(保管90日、以降自動削除)。
4. 顧客情報・人事情報を含む会議の録画は機密度「Restricted」設定。

## 4. キャンセルとノーショー
<!-- section_id: sec_4 -->

会議室の無駄占有を防ぐため、キャンセルポリシーを厳格に運用しています。

### 4.1 キャンセル手順

1. Google Calendarまたは Outlookで該当予定を開く。
2. **[削除]** をクリック。
3. 参加者に **[全員に通知]** を選択して送信。
4. 定例の場合 **[この予定のみ]** か **[今後すべての予定]** を選択。

### 4.2 15分ルール

会議開始から **15分間誰も入室しない** 予約は、Zoom Room センサー(動体検知+音声検知)により自動的にキャンセル扱いとなり、会議室は他者の予約に開放されます。

> **注意:** 15分以内に必ずいずれかの参加者が入室するか、入室予定があることを Calendar コメントに記載してください。

### 4.3 ノーショー累積管理

総務部は月次でノーショー回数を集計しています。

| 累積回数(月内) | 対応 |
| :--- | :--- |
| 1回 | システム通知のみ |
| 2回 | 上長宛にも通知 |
| 3回以上 | 予約権限を翌月1週間停止 + 総務部からヒアリング |
| 半期で6回以上 | 人事評価コメント対象 |

### 4.4 緊急キャンセル時の連絡

体調不良・トラブル等で開始直前にキャンセルする場合:

1. Calendar から削除(自動通知)。
2. Slackで参加者および総務部(#facility)へ一言連絡(「本日10時の富士、急遽キャンセルします」)。
3. 来客予定があった場合は受付(reception@sample.co.jp)へ即時連絡。

> **NG例:** Calendar上に予約を残したまま会議を実施せず、誰にも連絡しない → ノーショーカウント+次の利用者の業務阻害。
>
> **OK例:** 中止確定後すぐに Calendar 削除+ Slack 連絡 → 他チームが会議室を活用可能。

### 4.5 よくあるFAQ

> **FAQ:** Q. 会議が10分早く終わりました。次の予約者のために予約を短縮すべきですか?  A. はい。Calendar で終了時刻を実際の終了時刻に更新してください。会議室解放のメリットがあります。

> **FAQ:** Q. 取引先来訪のため会議室を予約しましたが、相手が30分遅れます。15分ルールに該当しますか?  A. Calendar コメントに「来客遅延、◯時◯分開始予定」と記載すれば自動キャンセル対象外です。
