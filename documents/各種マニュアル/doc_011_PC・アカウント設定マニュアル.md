## 1. PCセットアップ
<!-- section_id: sec_1 -->

株式会社サンプルでは、入社初日に IT部ヘルプデスク(本社4F)で業務用PCを受領します。配布PCはキッティング済み(初期OS設定・基本ソフト導入済み)ですが、本人による初期設定が必要です。

### 1.1 配布PCの標準スペック

| 職種 | 標準機種 | OS | スペック |
| :--- | :--- | :--- | :--- |
| 事務・営業 | Lenovo ThinkPad X1 Carbon | Windows 11 Pro | Core i5/16GB/512GB SSD |
| 開発・データ | MacBook Pro 14インチ | macOS Sonoma | M3 Pro/32GB/1TB SSD |
| デザイン | MacBook Pro 16インチ | macOS Sonoma | M3 Max/64GB/2TB SSD |

> **注意:** 機種選択は職種別に決まっています。希望機種への変更はIT部にて職務要件審査を経て判断します(申請から発注完了まで2週間)。

### 1.2 受領当日の初期設定手順(Windows)

1. PCを電源に接続し、起動。Microsoftログイン画面で配布時に渡された **初期ユーザー名(姓.名@sample.co.jp)** と **初期パスワード** を入力。
2. 初回ログイン後、強制的にパスワード変更画面に遷移。**12文字以上、英大文字・小文字・数字・記号を含む** ものを設定。
3. Windows Hello(顔認証または指紋認証)を設定。
4. **Microsoft Intune(MDM)** への登録ダイアログが表示されたら **[登録]** を選択(必須)。
5. BitLocker暗号化が自動開始。完了まで約30分(バックグラウンド)。完了は通知領域に表示。
6. **CrowdStrike Falcon Sensor** がタスクトレイで稼働していることを確認(青いアイコン)。
7. Windows Updateを実行し再起動。
8. 画面ロックを **5分自動** に設定(Intune ポリシー強制)。

### 1.3 受領当日の初期設定手順(Mac)

1. 電源投入後、自動的にDEP(Apple自動デバイス登録)が走り、**Jamf Pro(MDM)** への登録画面に遷移。
2. 配布されたApple ID(@sample.co.jp)とパスワードでサインイン。
3. FileVault暗号化を有効化(自動)。回復キーはJamfに自動退避。
4. Touch ID登録。
5. ソフトウェア・アップデートを最新まで適用。
6. CrowdStrike Falcon Sensorがメニューバーに常駐していることを確認。

### 1.4 セットアップチェックリスト(IT部提出)

| No. | 項目 | 完了確認方法 |
| :--- | :--- | :--- |
| 1 | 初期パスワード変更済 | 再ログインで確認 |
| 2 | MFA設定済(Microsoft Authenticator) | 認証アプリで承認確認 |
| 3 | BitLocker/FileVault有効 | 設定画面で確認 |
| 4 | CrowdStrike稼働 | アイコン表示確認 |
| 5 | 画面ロック5分 | スクリーンセーバー設定 |
| 6 | OS最新化 | 設定>更新で確認 |

> **NG例:** 初期パスワードのまま業務開始 → 情報セキュリティ規程違反、即時アカウントロック。
>
> **OK例:** 受領当日中に上記6項目を完了し、Slack #it-helpdesk に「セットアップ完了」と投稿。

### 1.5 紛失・盗難・故障時の連絡

| 事象 | 連絡先 | 緊急度 |
| :--- | :--- | :--- |
| 紛失・盗難 | IT部セキュリティ課(内線2299、夜間 080-XXXX-XXXX) | 即時(24時間) |
| ハードウェア故障 | IT部ヘルプデスク(内線2201) | 営業時間内 |
| ウイルス感染疑い | IT部セキュリティ課 + ネットワークから即切断 | 即時 |

## 2. アカウント発行
<!-- section_id: sec_2 -->

業務に必要なアカウントは、原則として入社初日にIT部から一括発行されます。追加SaaSの利用申請は本人がServiceNowから起票します。

### 2.1 自動発行アカウント一覧

| システム | 用途 | アカウント形式 | 発行タイミング |
| :--- | :--- | :--- | :--- |
| Google Workspace | メール・カレンダー・Drive・Meet | 姓.名@sample.co.jp | 入社日 |
| 社内Active Directory | 社内システムSSO | s12345(社員番号) | 入社日 |
| Slack | 全社チャット | 姓.名 | 入社日 |
| Notion | ナレッジ管理 | 姓.名@sample.co.jp | 入社日 |
| Box | ファイル共有 | 姓.名@sample.co.jp | 入社日 |
| KING OF TIME | 勤怠管理 | 社員番号 | 入社日 |
| Concur | 経費精算 | 姓.名@sample.co.jp | 入社日 |
| ServiceNow | IT申請窓口 | SSO | 入社日 |

### 2.2 申請制アカウント(職種別)

| システム | 主な利用職種 | 申請方法 | 月額ライセンス費 |
| :--- | :--- | :--- | :--- |
| Salesforce | 営業・CS | ServiceNow > SaaS申請 | 18,000円 |
| Jira/Confluence | 開発・PM | 同上 | 1,200円 |
| GitHub Enterprise | 開発 | 同上 | 4,800円 |
| Figma | デザイン・PM | 同上 | 2,250円 |
| freee会計 | 経理 | 経理部経由 | - |
| CloudSign | 法務・営業 | 法務部経由 | - |

### 2.3 アカウント申請手順(ServiceNow)

1. ServiceNow(sample.service-now.com)にSSOログイン。
2. トップページ **[サービスカタログ] > [SaaSアカウント申請]** を選択。
3. 申請テンプレ画面で以下を入力:
   - 申請者氏名(自動入力)
   - 利用部署(自動入力)
   - 申請システム名(プルダウン)
   - 利用目的(50字以上)
   - 上長承認者(直属上長)
4. **[送信]** をクリック。SLA:営業日48時間以内に開設。

### 2.4 MFA(多要素認証)設定

すべてのアカウントでMFAは必須です。入社時に **Microsoft Authenticator** アプリをスマートフォンにインストールしてください。

1. App Store または Google Play から「Microsoft Authenticator」を入手。
2. PCで aka.ms/mfasetup にアクセス。
3. 画面のQRコードをアプリでスキャン。
4. アプリに表示される6桁コードをPC画面に入力。
5. バックアップ用に電話番号(SMS)も登録。

> **注意:** スマートフォン紛失・機種変更時は、必ずIT部ヘルプデスクへ即時連絡してください。MFA未解除のまま放置するとロックアウトされ、業務停止します。

### 2.5 退職・異動時のアカウント処理

退職決定後、人事システム(SmartHR)から自動的にアカウント停止ワークフローが起動します。社員は退職日前日までに以下を完了:

1. Boxの個人領域ファイルを業務関連は引継先共有、個人は削除。
2. Slack DMの業務情報は引継書(Notion)に転記。
3. 私物の購入物・データは私用デバイスに移行(私的データの会社デバイス残置禁止)。

## 3. メール設定
<!-- section_id: sec_3 -->

株式会社サンプルの全社メールは Google Workspace(Gmail)です。Outlookでの利用も可能ですが、本ドキュメントではブラウザ版Gmailを基本とします。

### 3.1 メールアドレス命名規則

| パターン | 形式 | 例 |
| :--- | :--- | :--- |
| 個人 | 姓.名@sample.co.jp | tanaka.taro@sample.co.jp |
| 同姓同名重複時 | 姓.名2@sample.co.jp | tanaka.taro2@sample.co.jp |
| 部門共通 | 部門名@sample.co.jp | sales@sample.co.jp |
| プロジェクト | prj-プロジェクト名@sample.co.jp | prj-alpha@sample.co.jp |

社外向け署名は下記の形式に統一します。Gmail設定 > **[全般] > [署名]** に登録してください。

> **注意:** 趣味・SNSリンク・絵文字は社外向け署名に含めないでください。社内向け署名は任意に設定可です。

### 3.3 配信リスト(メーリングリスト)

| リスト名 | 用途 | 加入条件 | 投稿可能者 |
| :--- | :--- | :--- | :--- |
| all@sample.co.jp | 全社通知 | 全社員自動加入 | 役員・人事部のみ |
| info-tokyo@ | 本社拠点共通 | 本社所属で自動加入 | 全員 |
| sales-all@ | 営業本部全員 | 営業本部所属で自動加入 | 営業本部員 |
| dev-all@ | 開発本部全員 | 開発本部所属で自動加入 | 開発本部員 |
| pjt-XXX@ | プロジェクト個別 | PMが管理 | 参加者 |

### 3.4 自動転送・自動応答

| 機能 | 可否 | 備考 |
| :--- | :--- | :--- |
| 社外への自動転送 | **禁止** | 情報漏えいリスクのためDLPで全件ブロック |
| 社内アドレスへの自動転送 | 可 | ただし業務委任時のみ、上長承認推奨 |
| 不在自動応答(休暇等) | 可 | 期間・代理連絡先を明記 |
| 個人Gmailへの転送 | **禁止** | 違反は懲戒対象 |

```

代理連絡先:△△部 □□(yamada.hanako@sample.co.jp / 03-XXXX-XXXX)

休暇明け以降、順次ご返信させていただきます。
ご不便をおかけしますが、何卒よろしくお願い申し上げます。
```

### 3.6 機密度ラベル(社外送信時)

Gmailの **[ラベル付与]** から以下のいずれかを必ず選択します。

| ラベル | 用途 | 暗号化 |
| :--- | :--- | :--- |
| Public | 一般情報 | 不要 |
| Internal | 社内限定 | 不要 |
| Confidential | 関係者外秘 | TLS強制 |
| Restricted | 機密(個人情報・契約金額等) | S/MIME暗号化 |

> **NG例:** 顧客の個人情報を含む添付をPublicラベルで送信 → DLPで自動隔離、IT部監査対象。
>
> **OK例:** ConfidentialまたはRestrictedラベルを付与し、添付ファイルはBoxセキュアリンク共有に切替。

## 4. ソフトウェアインストール
<!-- section_id: sec_4 -->

業務に必要なソフトウェアは、原則として **会社管理ポータル(Windows:Company Portal / Mac:Self Service)** からインストールします。自己判断でのフリーソフト導入は禁止です。

### 4.1 標準導入済みソフト(キッティング時)

| ソフト | Win/Mac | 用途 |
| :--- | :--- | :--- |
| Google Chrome | 両方 | 業務標準ブラウザ |
| Google Workspace アプリ群 | 両方 | メール・Drive |
| Microsoft 365(Word/Excel/PowerPoint) | 両方 | Officeドキュメント |
| Outlook | 両方 | 補助メールクライアント |
| Slack デスクトップ | 両方 | チャット |
| Zoom / Microsoft Teams | 両方 | Web会議 |
| Box Drive | 両方 | ファイル共有 |
| Adobe Acrobat Reader DC | 両方 | PDF閲覧 |
| 1Password Business | 両方 | パスワード管理 |
| CrowdStrike Falcon | 両方 | EDR |

### 4.2 申請制ソフト(許可済み)

| ソフト | 主用途 | 申請先 | 標準納期 |
| :--- | :--- | :--- | :--- |
| Visual Studio Code | 開発 | Company Portal | 即時 |
| JetBrains IDE群 | 開発 | ServiceNow | 1営業日 |
| Adobe Creative Cloud | デザイン | ServiceNow | 2営業日 |
| Figma Desktop | デザイン | Self Service | 即時 |
| DBeaver | DB操作 | ServiceNow | 1営業日 |
| Postman | API検証 | Self Service | 即時 |
| Notion デスクトップ | ナレッジ | Self Service | 即時 |

### 4.3 申請手順(Windows例)

1. スタートメニューから **[Company Portal]** を起動。
2. 検索バーから希望ソフト名を入力。
3. 詳細画面で **[インストール]** をクリック。
4. インストール完了後、デスクトップに通知が表示。

### 4.4 申請手順(Mac例)

1. アプリケーション > **[Self Service]** を起動。
2. カテゴリ「Productivity」「Development」等から該当ソフトを選択。
3. **[Install]** をクリック → Touch ID認証 → 自動インストール。

### 4.5 許可外ソフト(原則禁止)

1. 個人ライセンスのソフト(Officeパーソナル等)
2. オンラインゲーム、ストリーミングアプリ(業務用途を除く)
3. ファイル共有P2Pソフト(BitTorrent等)
4. リモートデスクトップ系(Chrome Remote Desktop、AnyDesk等の個人利用)
5. 海外製の生成AIクライアント(社内ガイドライン未承認のもの)

> **注意:** 業務上どうしても必要な未承認ソフトがある場合は、ServiceNow > **[特例ソフト申請]** から起票し、IT部セキュリティ課のリスク評価(SLA:5営業日)を経て判断します。無断インストールはCrowdStrikeで検知され、IT部から即時連絡が入ります。

### 4.6 アップデート方針

| 種類 | 方針 |
| :--- | :--- |
| OSメジャーアップデート | IT部から一斉案内、Intune/Jamfで配布 |
| OSセキュリティパッチ | 自動適用(再起動は本人タイミング、48時間以内) |
| ブラウザ | 自動更新を有効化(必須) |
| 業務ソフト | 月次でCompany Portal/Self Serviceから手動更新 |

> **FAQ:** Q. 自宅のプライベートPCで業務をしてもよいですか?  A. 原則禁止です。やむを得ない場合は会社支給のVDI(Citrix)経由のみ可。BYOD許可申請を別途行ってください。
