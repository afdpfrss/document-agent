## 1. SMART原則
<!-- section_id: sec_1 -->

株式会社サンプルでは、すべての社員の年度目標はSMART原則に従って記述することを必須とする。SMARTとは、Specific(具体的)、Measurable(計測可能)、Achievable(達成可能)、Relevant(関連性)、Time-bound(期限明確)の頭文字で、目標の質を担保する世界標準のフレームワークである。HRBrainの目標設定画面には、入力時にSMART各要素のチェックリストが表示される。

### 1.1 Specific(具体的)
「営業力を強化する」のような抽象的目標は避け、対象顧客・施策・成果物を明示する。
- 悪い例:「営業活動を強化する」
- 良い例:「中堅製造業の新規顧客10社に対して、当社CRM製品の提案を行い、3社からPoC受注を獲得する」

### 1.2 Measurable(計測可能)
定量指標(KPI)を必ず添える。定性目標であっても、達成判定基準を「○○の状態が達成されている」と明文化する。
- 定量例:「新規アポイント月10件、商談化率30%、受注額3,000万円」
- 定性例:「四半期末までに新人2名がOJTチェックリスト全項目をクリアし、独力で顧客対応できる状態」

### 1.3 Achievable(達成可能)
リソース(時間・予算・人員・自身のスキル)を踏まえ、現実的に達成可能であるかを点検する。当社では「達成確率70%」を目安としたストレッチ目標を推奨しており、楽勝目標(達成確率95%以上)は原則認めない。

### 1.4 Relevant(関連性)
個人目標は、所属部門のKPI、本部方針、全社方針へとカスケードされて整合している必要がある。HRBrainの目標ツリー機能を用い、上位目標との紐付けを必ず登録する。紐付けのない目標は一次評価者が差し戻す。

### 1.5 Time-bound(期限明確)
通期目標であっても、四半期ごとのマイルストン(中間ゴール)を設定する。例:「4-6月:対象顧客リスト作成と訪問計画」「7-9月:提案実施」「10-12月:PoC実施」「1-3月:本契約・横展開」。マイルストン未達時の対応策も併記すると進捗管理が容易になる。

## 2. 目標数と難易度
<!-- section_id: sec_2 -->

### 2.1 目標数
- 業績目標(MBO):3〜5本
- 行動目標(コンピテンシー連動):2本
- 合計:5〜7本

目標数が多すぎると「全体最適化」より「個別タスク消化」に意識が向き、重要性の高い目標への集中が阻害される。逆に少なすぎると評価の信頼性が下がる。原則として5本前後を上限とし、超える場合は一次評価者の承認を要する。

### 2.2 難易度区分とウェイト
各目標には「難易度ランク」と「ウェイト(%)」を設定する。ウェイト合計は100%。

| 難易度 | 定義 | 達成時の評価係数 |
|---|---|---|
| ストレッチ | 過去実績の130%以上、または前例のない取組み | 1.3 |
| スタンダード | 期待水準どおり、達成可能性70% | 1.0 |
| ベース | 必達ライン、達成可能性90%以上 | 0.8 |

ストレッチ目標は最低1本含めることを推奨する。全目標がベースレベルの場合、行動評価で「主体性」「学習意欲」の項目が減点対象となる。

### 2.3 ストレッチ目標の定義
ストレッチ目標は以下のいずれかに該当するものを指す。
1. 過去最高実績を10%以上更新する数値目標
2. 当社で前例のない新規取組み
3. 他部門との横断的協働を要する取組み
4. 業務プロセスの抜本的改善(BPR)を伴う取組み

ストレッチ目標は未達でも評価係数1.0以上(達成率70%超なら按分加点)とする「失敗を許容する」設計とし、挑戦を促進する。

### 2.4 MBOとOKRの組合せ
当社では原則MBOを基本としつつ、新規事業部門および技術開発部門についてはOKR(Objectives and Key Results)を併用する。OKRは年度Objectiveを1〜3個、各Objectiveに対し3〜5個のKey Resultsを設定し、四半期ごとにレビューする。OKRの達成度合は60〜70%が健全とされ、100%達成はストレッチが不足していたと評価する。

## 3. 目標設定面談
<!-- section_id: sec_3 -->

### 3.1 事前準備
本人は目標設定面談に先立ち、以下を準備する。
1. 前年度の振り返り(自己評価ランク、達成/未達理由)
2. 上位目標(本部方針・部門KPI)の確認
3. 自身の目標案ドラフト(HRBrain下書き保存)
4. キャリア志向(中期キャリアプラン)
5. 必要な支援・リソース(研修・予算・人員)

一次評価者は、本人準備物の事前確認に加え、以下を準備する。
1. 部門KPIブレークダウン案
2. 本人の前年度評価所見
3. 本人に期待する役割定義(等級・職務記述書)
4. 期待行動レベル(コンピテンシー)

### 3.2 面談の進め方(60〜90分目安)
1. **冒頭(5分)** 面談の目的と進め方の確認
2. **前年度振り返り(15分)** 達成/未達の要因分析、学び
3. **上位目標カスケード(10分)** 本部・部門方針の共有
4. **目標案議論(40分)** SMART観点でひとつずつ精査
5. **支援事項合意(10分)** 上司・人事・他部門への依頼事項
6. **クロージング(5分)** 次回1on1の日程、登録期限の確認

### 3.3 認識合わせのコツ
目標設定面談での「合意形成」が評価期間中の納得感を左右する。以下を意識する。
- 上司は結論を先に押し付けず、本人の言葉を引き出す質問(オープンクエスチョン)を多用する
- 数値目標は根拠データを示し、「なぜその数字か」を双方で言語化する
- 困難度の認識ズレは、過去事例(類似プロジェクトの所要工数等)に照らして整合させる
- 合意できない場合は持ち帰り、3営業日以内に再面談を設定する

### 3.4 目標登録と承認
合意した目標は、面談から5営業日以内にHRBrainに本人入力、一次評価者が承認操作を行う。承認をもって「期初目標確定」とし、変更には所定の手続(後述)を要する。承認期限を過ぎた場合は人事部からアラートが配信される。

## 4. 進捗管理
<!-- section_id: sec_4 -->

### 4.1 月次1on1での進捗確認
目標進捗は月次1on1(隔週推奨)で確認する。HRBrainの進捗ステータス(順調/要注意/遅延)を本人が更新し、上司は「障害除去」の観点で支援する。1on1での合意事項はカオナビ1on1に記録し、評価エビデンスとして蓄積する。

### 4.2 四半期レビュー(7月・10月・1月)
四半期末には30分のレビューミーティングを実施し、以下を確認する。
1. 各目標の達成率(現時点)
2. 残期間で達成可能か(リスク評価)
3. 必要なリソース追加・支援
4. ストレッチ目標の到達見込み

四半期レビュー結果はHRBrain上に文章で残し、期末評価時の参照資料とする。

### 4.3 目標修正の条件と手続
以下のいずれかに該当する場合、期中での目標修正を認める。
1. 組織変更・異動により業務範囲が変わった場合
2. 担当顧客・担当製品の大幅な変更があった場合
3. 経営判断による戦略変更があった場合
4. 市場・規制環境の大幅な変化があった場合
5. 当初想定していなかった大型案件のアサインがあった場合

修正手続は以下のとおり。
- 本人がHRBrainで修正申請(変更理由必須)
- 一次評価者が内容を確認し承認
- 重要目標(ウェイト30%以上)の変更は二次評価者にも回付
- 修正履歴はシステム上に永久保存される

修正可能なのは9月中間レビュー時を原則とするが、緊急性が高い場合は随時受付ける。

### 4.4 記録方法
進捗記録は「事実(数値・成果物)+所感(本人見解)+上司コメント」の三層構造で残す。期末評価時に「何を、いつ、どう実行したか」を再現できる粒度を目指す。HRBrainでは月次入力テンプレートを用意しており、入力時間は1目標あたり5分以内を目安とする。

### 4.5 進捗管理に関する典型的な失敗と対策
| 失敗パターン | 原因 | 対策 |
|---|---|---|
| 期末まで進捗未更新 | 多忙・優先度低 | 月次1on1での更新ルール化 |
| 順調・順調・順調・期末未達 | リスク先送り | 四半期レビューでリスク強制ヒアリング |
| 目標と日常業務の乖離 | 目標が業務と無関係 | 目標設定段階での業務棚卸 |
| 修正乱発による評価困難 | 安易な変更 | 修正理由の必須化と承認の厳格化 |
